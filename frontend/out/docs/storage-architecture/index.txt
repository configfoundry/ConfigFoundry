3:T6418,<h1 id="storage-architecture">Storage Architecture<a href="#storage-architecture" class="heading-anchor" aria-label="Link to this section">#</a></h1>
<p>ConfigFoundry v0.5 introduces a <strong>Storage Abstraction Layer</strong> that completely decouples the application from any specific database driver or engine. No repository, service, or route contains database-specific logic; all of that lives inside a <code>StorageProvider</code> implementation.</p>
<hr />

<p>The <code>StorageFactory</code> selects the correct provider at startup based on config. Repos never import a driver directly — they receive a <code>StorageProvider</code> through constructor injection.</p>

<h2 id="storageprovider-interface">StorageProvider Interface<a href="#storageprovider-interface" class="heading-anchor" aria-label="Link to this section">#</a></h2>
<p>Every provider must implement all methods of the ABC:</p>
<div class="code-block"><div class="code-block-header"><span class="code-lang">python</span><button type="button" class="copy-btn" aria-label="Copy code">Copy</button></div><pre><code class="language-python">class StorageProvider(ABC):
    def initialize(self) -&gt; None: ...
        # Create engine, run migrations, seed data.
        # Raises NotImplementedError on scaffold providers.

    def shutdown(self) -&gt; None: ...
        # Dispose engine / close connections. Must be idempotent.

    def get_engine(self) -&gt; Engine: ...
        # Returns the live SQLAlchemy engine.
        # Raises RuntimeError if called before initialize() or after shutdown().

    def get_session_factory(self) -&gt; sessionmaker: ...
        # Returns a bound sessionmaker.
        # Raises RuntimeError if called before initialize().

    def health_check(self) -&gt; HealthCheckResult: ...
        # Non-raising. Returns HEALTHY/DEGRADED/UNHEALTHY with latency_ms.
        # Returns UNHEALTHY before initialize() and after shutdown().

    def get_metadata(self) -&gt; ProviderMetadata: ...
        # Works without a live connection (name, version, driver, dialect).

    def get_capabilities(self) -&gt; ProviderCapabilities: ...
        # Works without a live connection (feature flags).

    @property
    def name(self) -&gt; str: ...
        # Convenience — returns get_metadata().name.</code></pre></div>

<h2 id="dependency-injection">Dependency Injection<a href="#dependency-injection" class="heading-anchor" aria-label="Link to this section">#</a></h2>
<p><code>ServiceContainer</code> is the single wiring point. Every repository receives the <code>StorageProvider</code> via constructor injection:</p>
<div class="code-block"><div class="code-block-header"><span class="code-lang">python</span><button type="button" class="copy-btn" aria-label="Copy code">Copy</button></div><pre><code class="language-python"># container.py (simplified)

        self._provider = StorageFactory.create(config.database)
        self._provider.initialize()

        self.device_repo     = SQLAlchemyDeviceRepository(self._provider)
        self.bandwidth_repo  = SQLAlchemyBandwidthRepository(self._provider)
        # …

        self.device_service  = DeviceService(self.device_repo, self.audit_repo)

<p><code>ServiceContainer</code> also accepts <code>db_path: str</code> (SQLite shortcut, backward-compatible) and <code>provider: StorageProvider</code> (for tests that need direct control).</p>
<p>FastAPI resolves the container per request via <code>Depends(get_container)</code> in <code>api/dependencies.py</code>, which reads <code>request.app.state.container</code>.</p>

<h2 id="adding-a-new-provider">Adding a New Provider<a href="#adding-a-new-provider" class="heading-anchor" aria-label="Link to this section">#</a></h2>
<p>Follow these steps to add support for a new database backend:</p>
<p><strong>1. Create the provider module</strong> in <code>core/storage/providers/</code>:</p>
<div class="code-block"><div class="code-block-header"><span class="code-lang">python</span><button type="button" class="copy-btn" aria-label="Copy code">Copy</button></div><pre><code class="language-python"># core/storage/providers/duckdb.py
from core.storage.provider import (
    StorageProvider, HealthCheckResult, HealthStatus,
    ProviderCapabilities, ProviderMetadata,
)
from core.storage.config import DatabaseConfig

class DuckDBProvider(StorageProvider):
    def __init__(self, config: DatabaseConfig) -&gt; None:
        self._config = config
        self._engine = None
        self._session_factory_obj = None

    def initialize(self) -&gt; None:
        from sqlalchemy import create_engine, event
        from sqlalchemy.orm import sessionmaker

        url = f&quot;duckdb:///{self._config.sqlite_path}&quot;   # or connection_url
        self._engine = create_engine(url, **self._extra_kwargs())
        self._session_factory_obj = sessionmaker(bind=self._engine)
        # run any migrations here

    def shutdown(self) -&gt; None:
        if self._engine:
            self._engine.dispose()
            self._engine = None

    def get_engine(self):
        if self._engine is None:
            raise RuntimeError(&quot;DuckDBProvider.initialize() has not been called&quot;)
        return self._engine

    def get_session_factory(self):
        if self._session_factory_obj is None:
            raise RuntimeError(&quot;DuckDBProvider.initialize() has not been called&quot;)
        return self._session_factory_obj

    def health_check(self) -&gt; HealthCheckResult:
        import time
        if self._engine is None:
            return HealthCheckResult(HealthStatus.UNHEALTHY, &quot;Not initialised&quot;)
        try:
            from sqlalchemy import text
            t0 = time.perf_counter()
            with self._engine.connect() as conn:
                conn.execute(text(&quot;SELECT 1&quot;))
            latency_ms = (time.perf_counter() - t0) * 1000
            return HealthCheckResult(HealthStatus.HEALTHY, &quot;OK&quot;, latency_ms=latency_ms)
        except Exception as exc:
            return HealthCheckResult(HealthStatus.UNHEALTHY, str(exc))

    def get_metadata(self) -&gt; ProviderMetadata:
        return ProviderMetadata(
            name=&quot;duckdb&quot;, version=&quot;unknown&quot;, driver=&quot;duckdb_engine&quot;, dialect=&quot;duckdb&quot;,
            capabilities=self.get_capabilities(),
        )

    def get_capabilities(self) -&gt; ProviderCapabilities:
        return ProviderCapabilities(
            transactions=True, migrations=True, full_text_search=True,
            json_columns=True, array_columns=True, schemas=True,
        )

    def _extra_kwargs(self) -&gt; dict:
        return {&quot;echo&quot;: self._config.echo, **self._config.connect_args}</code></pre></div>

<h2 id="scaffold-providers">Scaffold Providers<a href="#scaffold-providers" class="heading-anchor" aria-label="Link to this section">#</a></h2>
<p><code>PostgreSQLProvider</code>, <code>MySQLProvider</code>, and <code>SQLServerProvider</code> are <strong>interface-compliant scaffolds</strong>:</p>
<ul><li><code>initialize()</code> → raises <code>NotImplementedError</code> with a helpful message</li><li><code>get_engine()</code> / <code>get_session_factory()</code> → raise <code>RuntimeError</code> (no engine available)</li><li><code>health_check()</code> → returns <code>UNHEALTHY</code> with <code>{&quot;scaffold&quot;: True}</code> in <code>details</code> — never raises</li><li><code>shutdown()</code> → no-op</li><li><code>get_metadata()</code> / <code>get_capabilities()</code> → fully implemented, work without a live connection</li></ul>
<p>This means scaffold providers can be inspected and registered without any database being present, making feature-detection and configuration UIs possible even before a backend is implemented.</p>
<hr />
<h2 id="session-management-strategy">Session Management Strategy<a href="#session-management-strategy" class="heading-anchor" aria-label="Link to this section">#</a></h2>
<p>All repositories use <strong>per-operation sessions</strong> via a context manager:</p>
<div class="code-block"><div class="code-block-header"><span class="code-lang">python</span><button type="button" class="copy-btn" aria-label="Copy code">Copy</button></div><pre><code class="language-python">with Session(self._engine) as session:
    # read or write
    session.commit()</code></pre></div>
<p>This keeps repositories stateless (no held connections between calls), makes them safe to reuse across requests without locking, and allows the module-level storage shim to work without FastAPI request context.</p>
<p><strong>No shared session is passed between repos or services.</strong> Each repo call opens and closes its own session. This is the recommended pattern for SQLAlchemy 2.x write workloads with a connection pool.</p>

<h2 id="technical-debt--next-milestone-recommendations">Technical Debt &amp; Next-Milestone Recommendations<a href="#technical-debt--next-milestone-recommendations" class="heading-anchor" aria-label="Link to this section">#</a></h2>
<h3 id="high-priority">High priority<a href="#high-priority" class="heading-anchor" aria-label="Link to this section">#</a></h3>
<ul><li><strong>Implement PostgreSQL/MySQL/SQL Server providers.</strong> The scaffolds are fully wired — only <code>initialize()</code>, <code>get_engine()</code>, and <code>get_session_factory()</code> need real bodies. Add <code>psycopg2</code>, <code>pymysql</code>, and <code>pyodbc</code> to <code>requirements.txt</code>, and adapt <code>SQLiteProvider._run_migrations()</code> to use SQLAlchemy DDL or Alembic instead of raw <code>sqlite3</code> commands.</li></ul>
<ul><li><strong>Migrate to Alembic.</strong> The current migration system uses raw <code>sqlite3</code> statements in <code>core/migrations.py</code>. Alembic would make schema evolution provider-agnostic and give a proper revision history.</li></ul>
<ul><li><strong>Async support.</strong> FastAPI + SQLAlchemy 2.x support <code>AsyncEngine</code> and <code>AsyncSession</code>. Adding an <code>AsyncStorageProvider</code> subtype (or a flag on the existing ABC) would allow fully async request handling.</li></ul>
<h3 id="medium-priority">Medium priority<a href="#medium-priority" class="heading-anchor" aria-label="Link to this section">#</a></h3>
<ul><li><strong>Connection URL validation.</strong> <code>DatabaseConfig</code> accepts any string as <code>connection_url</code>. Add a <code>validate()</code> method (or Pydantic model) that checks the URL scheme matches the declared provider before <code>initialize()</code> is called.</li></ul>
<ul><li><strong>Pool health monitoring.</strong> Expose <code>pool.status()</code> in <code>health_check()</code> details so operators can see checkout counts, overflow, and timeouts without hitting the DB directly.</li></ul>
<ul><li><strong>Config hot-reload.</strong> Currently the provider is created once at startup. Introducing a reload endpoint (or SIGHUP handler) that recreates the container without restarting the process would help in long-lived deployments.</li></ul>
<h3 id="low-priority">Low priority<a href="#low-priority" class="heading-anchor" aria-label="Link to this section">#</a></h3>
<ul><li><strong>Remove <code>core/db.py</code>.</strong> This file was the pre-abstraction entry point for <code>init_db()</code>. It is no longer called by any production code (SQLiteProvider owns that logic). It can be deleted once any remaining direct imports are confirmed absent.</li></ul>
<ul><li><strong>Drop the backward-compat shim functions</strong> in <code>core/storage/__init__.py</code> (<code>list_devices()</code>, <code>upsert_device()</code>, etc.) once the handler tests are updated to call the container directly.</li></ul>
