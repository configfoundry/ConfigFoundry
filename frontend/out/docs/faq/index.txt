<p><strong>Does ConfigFoundry require internet access to run?</strong> No. Install and runtime are both fully offline-capable — see <a href="/docs/airgap/">Air-Gap Deployment</a>. Only regenerating the offline vendor bundles (when bumping a dependency version) needs internet access, and that&#39;s a build-time step, never a runtime one.</p>
<p><strong>Do I need Node.js installed?</strong> Only if you&#39;re rebuilding the frontend from source. The release bundle and a normal git clone both include a pre-built <code>frontend/out/</code> static export, so most installs never touch Node. See <a href="/docs/installation/">Installation</a>.</p>
<p><strong>What database should I use?</strong> SQLite by default — zero configuration, one file, fine for small and mid-sized teams. Move to PostgreSQL for larger or multi-instance deployments. See <a href="/docs/storage/">Storage</a>.</p>
<p><strong>Is multi-tenancy supported?</strong> The security layer (users, roles, API keys, policies, audit) is fully organization-scoped and ready for it. The inventory tables (devices, bandwidth, subnets, etc.) are not yet retrofitted with per-organization scoping. See <a href="/docs/enterprise/#multi-tenancy">Enterprise § Multi-tenancy</a>.</p>
<p><strong>How do I get API access for a script or collector, without a human login?</strong> Create an API key (<code>POST /api/v1/api-keys</code>, requires <code>api:manage</code>) — it can be scoped to a specific permission list, IP allowlist, and expiration. Use it as <code>Authorization: Bearer cfk_live_...</code>. See <a href="/docs/authentication/">Authentication</a>.</p>
<p><strong>Is MFA required?</strong> Not by default. Set <code>CONFIGFOUNDRY_AUTH_MFA_REQUIRED_ROLES</code> to enforce it for specific roles (Super Admin and Organization Admin are strongly recommended at minimum for anything beyond a personal instance). See <a href="/docs/configuration/">Configuration</a>.</p>
<p><strong>How do I reset my own password?</strong> <code>POST /api/v1/auth/password/change</code> while logged in. If you&#39;ve lost access entirely, another admin can reset it via <code>POST /api/v1/users/{id}/reset-password</code> — see <a href="/docs/troubleshooting/">Troubleshooting</a>.</p>
<p><strong>Can I run multiple instances behind a load balancer?</strong> Not validated for SQLite (single-writer limitation). With PostgreSQL as the shared backend, multiple instances can point at the same database, though there&#39;s no built-in session affinity requirement — the JWT/refresh-token model doesn&#39;t need sticky sessions. Rate limiting is per-process, so each instance enforces its own independent limit rather than a shared global one. See <a href="/docs/deployment/">Deployment</a>.</p>
<p><strong>Where do generated YAML configs go?</strong> <code>POST /api/v1/generate</code> returns the generated config in the response and records it in <code>history</code> (<code>GET /api/v1/history</code>) — nothing is written to disk automatically. Save the response wherever your deployment pipeline expects it.</p>
<p><strong>How is this different from just using a shared spreadsheet?</strong> Multi-user concurrent access to one dataset (no per-person copies to reconcile), input validation, an audit trail of every change, and generated YAML that&#39;s always derived from current data rather than hand-copied. See the README&#39;s &quot;Why this exists&quot; section.</p>
<p><strong>Does ConfigFoundry send any data externally — telemetry, analytics, update checks?</strong> No. There is no telemetry, analytics beacon, or phone-home update check of any kind. This is enforced by the same air-gap validation that checks for CDN/external-asset references — see <a href="/docs/airgap/">Air-Gap Deployment</a>.</p>
<p><strong>How often are dependencies updated?</strong> No fixed cadence currently — see <a href="/docs/release-process/">Release Process</a> for how a release is cut and <a href="/docs/roadmap/">Roadmap</a> for what&#39;s planned.</p>
<p><strong>I found a security issue. Where do I report it?</strong> See <a href="/docs/security/#reporting-a-vulnerability">Security § Reporting a vulnerability</a>.</p>
<p><strong>Can I add a new database provider / new integration?</strong> Yes — both are designed as extension points. See <a href="/docs/storage/#adding-a-new-provider">Storage § Adding a new provider</a> and <a href="/docs/architecture/#layering-rules">Architecture § Layering rules</a> for why integrations depend on core and never the reverse.</p>5:["slug","faq","d"]
