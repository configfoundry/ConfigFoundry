# ConfigFoundry — runtime dependencies (pinned, exact versions)
#
# Every version below is pinned exactly (no ranges) so that:
#   1. `vendor/python/` (the offline wheelhouse — see docs/airgap.md) always
#      contains precisely what a fresh install resolves to. A range like
#      `fastapi>=0.100.0` can silently resolve to a *different* release on
#      a machine with internet access than what's vendored, which would
#      defeat the point of the wheelhouse.
#   2. Air-gapped installs via `pip install --no-index --find-links
#      vendor/python -r requirements.txt` are fully reproducible.
#
# Deliberately NOT using `fastapi[standard]`: that extra pulls in
# fastapi-cloud-cli, sentry-sdk, jinja2, python-multipart, and
# email-validator, none of which this application uses (no HTML templates,
# no file-upload forms, no telemetry/cloud reporting -- and definitely no
# outbound SDK that "phones home" in a product meant for bank/defense/
# air-gapped deployments). Only `uvicorn[standard]` is kept as an extra,
# for uvloop/httptools/watchfiles (real performance + dev-reload wins).
#
# To regenerate this file and the wheelhouse after a deliberate dependency
# bump, see "Updating vendored dependencies" in docs/airgap.md -- the short
# version is: bump the version(s) below, then re-run
# `scripts/build_python_wheelhouse.sh`.
