## v0.9.x — Beta

API, configuration format, and installer all frozen — no breaking changes accepted during this phase. The point of the freeze is community testing against something stable enough to be worth testing.

## v1.0.0 criteria

All of the following must be true, in full, before a v1.0.0 tag:

- **Stable API** — no breaking `/api/v1/` changes for a full release cycle.
- **Stable database schema** — migrations only, no manual intervention required.
- **Stable installer and upgrade path** — proven across real upgrades, not just fresh installs.
- **Real production deployments** running it.
- **Community feedback** from the v0.9.x beta incorporated.

Not a fixed date — a checklist. See [[Roadmap Overview#Versioning strategy]].

## What changes at v1.0.0

Per [[Development/Engineering Wiki#Versioning|Engineering Wiki § Versioning]]: pre-1.0, minor version bumps may include internal breaking changes even though `/api/v1/` itself stays stable. Post-1.0, the semantic versioning contract tightens for the whole project, not just the API surface.
